# EtereCitizen Protocol — Specification Artifacts

This directory contains the machine-readable artifacts that accompany the normative [protocol specification](../SPEC.md):

```
spec/
├── schemas/               # JSON Schema Draft 2020-12 files for every payload.
└── test-vectors/          # Deterministic input/expected pairs a conforming
    ├── did-parsing/       #    implementation MUST produce identical results for.
    ├── trust-score/
    ├── temporal-decay/
    ├── antifraud/
    └── challenge-serialization/
```

## Schemas

Every JSON Schema in `spec/schemas/` corresponds to a numbered section of SPEC.md. Implementations MAY validate payloads against these schemas; the schemas are declaratively authoritative on shape, and the normative prose in SPEC.md is authoritative on behavior.

## Test Vectors

Each vector is a JSON document with two top-level keys:

- `input`: the exact inputs a function receives.
- `expected`: the exact output a conformant implementation must produce.

Vectors are numbered. Additional vectors MAY be added over time; conforming implementations MUST pass **all** vectors in each subdirectory.

### Provenance

Vectors in this directory were generated by running the reference TypeScript implementation in `packages/sdk/` and `packages/common/` and capturing the exact outputs. When the specification diverges from the implementation, the specification is authoritative — implementers SHOULD report divergences as issues.

### Floating-point

Trust score and temporal decay values are double-precision floats. Implementations MUST match the expected value to at least 12 significant decimal digits. If an implementation produces a value that disagrees only in the last few bits, it SHOULD document its arithmetic rationale; larger divergences are non-conformant.
